📑 愿景：解决AI创作的“长篇终结”痛点

目前的 AI 写作工具大多停留在“扩写”或“单章生成”层面，随着字数增加，必然面临设定污染、逻辑崩塌、人设失真（OOC）、伏笔断裂等致命问题。

NovelEngine 不是一个简单的提示词集合，而是一套基于 多代理接力（Multi-Agent Relay） 和 动态约束系统（Dynamic Constraint System） 的小说生产协议。我们致力于将不可控的灵感闪现，转化为可预测、可回退、可审计的工业化流水线。

🚀 核心优势：为什么我们能写出“不降智”的长篇？
1. 人物五层模型：一致性引擎

不同于传统的人设标签，我们构建了“角色如何变化仍然还是同一个人”的动态系统。通过哲学起点（第0层）到冲突回写（第4层）的演化路径，确保角色在经历百万字洗礼后，其成长轨迹依然逻辑自洽。

2. 多代理非全知博弈：主角“被动生成”机制

我们拒绝 AI 开启“全知视角”强行推剧。系统中每个角色由独立 LLM 实例扮演，仅持有属于该角色的最小必要信息包。主角的选择不是被作者安排的，而是在配角、反派及世界规则的层层博弈中，被“逼”出来的唯一解。

3. 世界规则锁与场景执行器

我们将世界观转化为物理级的“操作系统”。每场戏开始前，执行器会激活当前场景的规则锁。如果世界设定“魔法消耗精神力”，AI 绝无法在精神力耗尽时强行开挂，规则不再是背景板，而是影响剧情走向的真实力量。

4. 情绪与情节的双螺旋上升

通过参数化控制（爽点密度、情绪波幅、兑现延迟），我们实现了剧情推进与读者情绪的精确对齐。系统不仅记录“发生了什么”，更在宏观与微观层面确保“读者爽不爽”。

5. 伏笔账本（Ledger）与精准激活

解决长篇网文“埋了忘，收了突兀”的痼疾。系统自动记录伏笔的生命周期，根据剧情节奏提示主编在最佳窗口期进行“跨章节价值注入”，实现草蛇灰线的艺术质感。

6.量化、数据、提示词多维迭代

不仅针对模型，也针对市场、系统，让他技能化、可扩展可自迭代。

🛠️ 六层架构：生产流程全景图

我们的方法论将小说生产拆解为六个相互约束的阶段：

世界模型层 (World Model)：定义底层物理规则、势力事实包，建立信息揭露依赖图。

剧情规划层 (Plot Planning)：L1A级大纲规划，锁定商业卖点与情绪主脉络。

推演细纲层 (Deduction Outline)：多代理博弈推演，生成具备冲突压力的节拍细纲。

正文生成层 (Text Generation)：多工位协作渲染。场景执行器设定边界，文学写手兑现风格指纹。

审计优化层 (Audit & Optimization)：客观逻辑（硬门槛）+ 主观商业（建议层）双轨审计，主编裁决回退或放行。

知识沉淀层 (Knowledge Accumulation)：真值回填 PgSQL，更新角色状态快照，为下一章构建隔离信息包。

💎 数据驱动与控制逻辑

唯一真值层 (PgSQL)：存放所有已发生的事实、人物状态与账本数据。

语义参考层 (Vector DB)：存放世界设定、风格指南、梗库等可语义检索资产。

工程控制模式：不同于“写完再改”的反馈模式，我们采用“参数输入 -> 场景推演 -> 差分审计”的闭环控制，确保输出即达标。
